adelie

A slideshow program

Adelie is a plain-text presentation tool and format. It is used to create slideshows with uxn. It features a handful of commands to quickly place text and images on slides, it can be operated with a keyboard, mouse or a game controller.

The program features two Uxf fonts to choose from, and TGA image support. All sizes are in hexadecimal numbers.

Getting Started

You must have the uxn assembler and emulator to use Adelie.

uxnasm src/main.tal bin/adelie.rom && uxnemu bin/adelie.rom

If do not wish to assemble it yourself, you can download adelie.rom.

See the project repository.

Commands

Sideshow files contain one command, or comment, per line.

size
set slide size
name
set slide name
goto
move to absolute position
mode
set text color
fill
fill background
pict
draw picture
head
draw header
text
draw text
link
make link region

Controls

mouse1
next slide
mouse2
draw marker
mouse3
prev slide
arrow + →
previous slide
arrow + ←
next slide
alt
previous slide
ctrl
next slide
shift
toggle link view

Making a slideshow

Create a .txt file, for example myproject.txt.

The project text file consists of a series of blocks of text, one 'block' per slide.

Below is an example of an all-text slide (HEAD):

NAME Slide 1
FILL 00
GOTO 03,24
MODE 0f
HEAD Larger Header
MODE 08
MOVE 00,05
HEAD Smaller text`with a linebreak

And here is a slide with both text and an image(PICT):

NAME Slide 2
GOTO 00,00
PICT picture.tga
MOVE 30,02
MODE 0a
HEAD Picture name

Hexadecimals

Adelie uses hexadecimals. In hexadecimals, the numbers 0-9 are the same but the numbers 10-15 become the letters A-F (11[b],12[c],13[d].14[e],15[f]). So if the desired text position of an image in decimals is 12,11 the hexadecimal equivalent would be c,b.

The decimal numbers 16-25 represent the hexadecimal numbers 10-19 (16[10],17[11],18[12]..). From 26-31, the numbers become adjoined by letters. For example, 26-31 become 26[1a], 27[1b],31[1f] etc. For the rest, refer to the below table.

Size

The default ratio for the project is 768x448(30,1c). Specifying a size in the project text file (at the top of the text file before the slides), will load the slideshow at that resolution. When deciding on a slide size, it is necessary to use multiples of 8.

Example:

SIZE 30,1c ( 768 x 448 )

If, for example, the desired size is 720x480, it is necessary to make a calculation to get the correct size for Adelie. 720 and 480 must each be divided by 16, and then converted to hexadecimals.

Example:

720/16=45
Converted to hexadecimals 45 turns into 0x2d.
480/16=30
Converted to hexadecimals 30 turns into 0x1e.

In the project file the text would be written as follows, with lowercase letters only and omitting the two first characters 0x:

SIZE 2d,1e

Goto

The command GOTO is used to position both text and images in a slideshow.

In the project text file, the GOTO command must sit on the line right above the text or image it will affect.

Example:

GOTO 24,04
HEAD We began using`technology as little`as possible.

Mode

Adelie uses hexadecimal colors (hex triplets), set with the MODE command. Each slide can have its own text color.

Setting MODE to 0a(black), the hex color red value is 0, green value is 0 and the blue value of its RGB is 10(a in hexadecimal). Red would be f00, or 100% red, 0% green and 0% blue etc.

Fill

The command FILL will insert a solid background to the slide. Enter a color in hexadecimal.

If an image takes up the entire canvas, a fill is not needed, but if the image is repositioned on the canvas the previous slide will be visible underneath. Adding a fill will keep that from happening.

Pict

The command PICT is used to add images in the slideshow.

Adelie requires that images be saved in the .tga format. When exporting images to .tga, uncheck the RLE for compression box (if it option comes up), and change Bottom left in the Origin drop-down menu to Top left.

By default, images will be centred on the canvas(00,00) and can be re-positioned using GOTO following by a hexadecimal position.

Head and text

Headers are labelled as HEAD, and sub-text as TEXT. Header text is larger, while sub-text is smaller.

Writing text without linebreaks ` will leave the text as a continous phrase, which may spill out of the canvas. It is not necessary to add spaces before or after the `.

Below is the text used to make the above slide:

NAME Power solutions
FILL 03
GOTO 04,03
PICT weeding.tga
GOTO 04,04
HEAD Doing away with`bloated toolchains.
GOTO 08,10
TEXT Weed out sources of waste.

Testing the project

To run the slideshow, in a terminal window navigate to the project folder and type:

uxnemu bin/adelie.rom yourfilename.txt

Example file

Here is an example of a 4 slide project:

SIZE 2d,1e

NAME logo
GOTO 00,00
PICT media/nime.tga

NAME 100r logo
FILL 03
GOTO 00,00
PICT media/100r.tga

NAME Rek and Dev
FILL 03
GOTO 00,00
PICT media/rekdev.tga

NAME Setup
FILL 03
GOTO 0d,00
PICT media/sailing.tga
GOTO 04,05
MODE 05
HEAD Living & working`from a sailboat.
GOTO 08,12
TEXT Name: Pino`Make: Yamaha`Length: 10m`Made: 1982`Origin: Shizuoka, Japan
