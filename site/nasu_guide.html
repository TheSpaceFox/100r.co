Launching Nasu

Nasu is distributed as a rom for the uxn system. In the terminal navigate to the folder in which you want your project files to be saved, then enter the following to launch Nasu rom:

~/roms/nasu.rom

Re-naming a file

The default filename reads untitled.chr. To change it, click on the text. It will flash white, click on the text field and erase the existing filename. Enter the new filename with the .chr extension. Always include the extension when re-naming a file.

Read more about the chr file format.

Saving a file

Click on the white star located on the lower right of the canvas.

The star has two states: Grey means the file was saved. White means that the file was changed and that the change has not yet saved

The state change makes it easy to know if you still have unsaved changes.

Importing a file

To import an file, erase the current filename and write the name of the .chr file you wish to import, and click enter. To import the file click on the checkered square next to the star (save) icon. The checkered square will import your file.

If switching between projects, close Nasu and navigate to the correct project folder in the terminal prior to launching it again.

New file

To load a blank file, click on the empty square next to the checkered square icon.

Be sure to re-name your file.

Blend view

In the Blend View there are 4 rows, and 4 columns, which correspond to all of the color possibilities in a project.

If you have a tile with pixel information selected, the blend view will display all the color possibilities. Clicking on one will alter the tiles on your canvas. Each row has 4 options for colors that you can choose from. Selecting the space on the first column of the first row will change your tile to tone on tone, if you select the second space on that row it changes the color arrangement.

Pre-view

The second top menu may appear cryptic, and will only be useful when creating spritesheets.

Typically, when preparing files for a project all of the tiles for a given sprite will need to be lined up from left to right. For someone assembling a character than spans multiple titles for a game, it is easier to locate the position of each piece if they follow each other. The issue with having many tiles lined that way, is that if they form a sprite that spans many tiles, it is difficult to make sense of it. The pre-view permits you to view the whole sprite by selecting the number of tiles it is meant to fill.

Say your sprite is 16x16 pixels, or 2x2 tiles, on the canvas the 4 tiles will follow each other horizontally. To visualize them as a full sprite, select the starting tile for that sprite with the select(arrow) tool. Pieces appear in the pre-view, if you navigate to that window and select 2x2 tiles, the asset will appear as it ought to look when assembled. The number below indicates the tile size, 2x2 or 22 for our example project

Color view

Color view is located on the top bar, third menu. This view permits you to change colors. There are 3 opaque sliders on a checkered background, moving the sliders will change the colors in Nasu.

The number underneath the sliders is the color code for the project. When making a game, write down this number so you can replicate the color scheme of the project.

Zoom view

When selecting a tile with pixel information it, you'll notice that an enlargened version of it appears on the fourth menu on the top bar. This is an isolated view of your tile that you can use to make quick changes (you can draw it in, and it will change the version on the canvas).

The up and down arrows permit you to offset the tile, say if you need to preview an asset that needs to tile perfectly with another. It's also useful if you want to move a shape up and down by a few pixels without having to re-draw it.

Data view

The data view menu is located on the upper right corner of the window, if your canvas is empty it will read 4 rows of 8 zeros, with a row of 4 zeros underneath. Data view, as its name suggests, is a way to rapidly view the code for each asset.

The numbers in row of 4 numbers change as you click around in the empty canvas, marking the position of your cursor. This is useful to identify and locate where sprites start and end in a .chr file. If you click on the first upper right tile, the number will read oooo. Moving from left to right this number will increase until it starts to use letters. This is because uxn uses base-16, a way to identify byte(8 bits form one byte) values without having to look at 1's and 0's. It's similar to the number system we use everyday to count with (decimal), in the way that it combines a set of digits to create large numbers (1 paired with a 0 is 10). Like hexadecimal (colors), the language in Uxn uses a set of 16 unique digits. Base-16 uses the standard 0-9 numbers, like the decimal system, but it also incorporates letters: A, B, C, D, E, and F.

The table below shows the letters and numbers used in base-16, and their corresponding collection of 1's and 0's in binary:

000004010081000C1100
100015010191001D1101
2001060110A1010E1110
3001170111B1011F1111

It isn't crucial to understand this to use Nasu, it is written here in the interest of sharing what things work and what every number or letter represents.

When drawing pixels in the canvas, numbers and letters will appear in the upper 4 rows of 0's. If you save your file, and open the .chr file in a text editor, the numbers and letters will match the ones in your data view. Making sense of a .chr document is tricky, but the data view can help locate specific tiles.

If you fill the canvas with a drawing, the information in the data view will change with letters and numbers. This is your drawing in numbers and letters. Copying this sequence of numbers and letters in a new .chr file would result in the same pixel arrangement.

Color tool

At the bottom left, there are 3 colored circles. You can only ever use 4 colors(including background) in a project. Change color by selecting a circle. Unselecting all circles will default to black, which by default is the background color. Black assigned as background color doubles as an eraser. Right-click on a pixel in your canvas to 'erase' it.

Pencil tool

The pencil tool is next to the select tool, used to draw pixels on the canvas.

No further explanation is needed, we think :).

Select tool

The 'select' tool is the arrow, located next to the color tool.

The select tool is used to move between tiles, it's also useful to copy tiles from one place on the canvas to another. To do this, select the tile you wish to duplicate, move elsewhere and right-click. The tile will appear on the new spot as a duplicate. This is a useful tool when building spritesheets.

Zoom tool

For a more detailed view of your drawing, select the 'magnifying glass' tool located on the right of the select tool.

This view shows outlines of each 8x8 tile, with each pixel appearing as a dot. This tool is useful to make detailed corrrections to a tile. Click on the magnifying glass again to return to the normal view.
